# Submission Checklist

- Project has a public URL or a clearly documented local-only fallback.
- `README.md` explains setup, run, test, and architecture.
- `.env.example` documents all required variables.
- Main demo flow is written down step by step.
- Rubric criteria are mapped to features or evidence.
- Screenshots or demo evidence exist for major flows.
- Known limitations are documented honestly.
- Future work is separated from missing required work.
- Repo can be run from a clean checkout.
- Final branch/worktree is in a reviewable state.
