# TalTech Computers

## Focus areas

- architecture basics
- memory and storage implications
- performance-aware reasoning
- how hardware constraints affect code and deployment

## Expectations

- connect software behavior back to CPU, memory, storage, and I/O
- explain performance and capacity issues concretely
- avoid vague statements like "the machine is slow" without a cause model

## Good output should include

- a clear explanation of the relevant hardware concept
- how it affects the program or system
- one concrete example or implication

## Common mistakes

- confusing storage speed with RAM behavior
- discussing performance without naming the bottleneck
- using hardware terms imprecisely
- ignoring architecture constraints in deployment choices

Do not treat system behavior as magic. Explain the machine-level consequences.
