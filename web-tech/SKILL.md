# TalTech Web Tech

## Focus areas

- HTTP and request flow
- semantic HTML
- CSS that is stable and maintainable
- browser-side JavaScript behavior
- client/server boundaries

## Review checklist

Review for:

- broken document semantics
- inaccessible forms and controls
- fragile layouts
- weak separation between content, styling, and behavior
- misunderstood browser or HTTP behavior

## Expected output

Good guidance or review should name:

- the specific browser or web-platform concept involved
- what the page or app currently does
- why that behavior is correct or incorrect
- the simplest standards-based fix

## Standard

Prefer standards and simple architecture over clever front-end machinery.
If a web concept can be explained with the platform itself, do that before introducing a library.
