# T-AI-LTech Stack

This file describes the TalTech-specific workflow layer used by T-AI-LTech.

## Purpose

The goal is not generic startup execution.
The goal is to support TalTech `IADB17` coursework across languages and course topics with strong code quality expectations.

## Workflow

```text
assignment or idea
    |
    v
/intake
    |
    v
scope + course fit + stack choice
    |
    v
/plan
    |
    v
architecture + test scope + boundaries
    |
    v
/build
    |
    v
implementation + tests + docs
    |
    v
/code-quality
    |
    v
maintainability review + bug review
    |
    v
/deploy
    |
    v
deployment or demo-ready local setup
    |
    v
/demo
    |
    v
evidence + screenshots + smoke checks
    |
    v
/submit
    |
    v
submission package + requirement mapping
```

## Standard

Every TalTech project should converge toward:

- a course-appropriate stack
- maintainable code
- automated tests on the critical path
- a clean setup story
- a stable demo path
- submission-ready documentation
