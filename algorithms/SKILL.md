# TalTech Algorithms

## Focus areas

- correctness
- time and space complexity
- choosing the right data structure
- edge cases and proof of reasoning

## Expectations

- state assumptions about input size and constraints
- justify the chosen algorithm or data structure
- explain complexity, not just the final answer
- test edge cases intentionally

## Review checklist

When reviewing, call out:

- unnecessary complexity
- hidden quadratic behavior
- incorrect edge-case handling
- data structures that are mismatched to the problem

## Expected output

Good output should state:

- the chosen algorithm or structure
- why it fits the input constraints
- time and space complexity
- the edge cases that matter most
